Introducción docker Imagenes y contenedores Docker-compose

¿Qué es un imagen DOCKER?

Una imagen es una plantilla, la cual será la base para crear los contenedores. Las imágenes pueden estar en local o en el repositorio de docker(). Si se quiere crear una imagen personalizada es necesario crear un Dockerfile. Por ejemplo una imagen podría contener un sistema operativo Ubuntu con un servidor Apache y tu aplicación web instalada.

¿Qué es un contenedor DOCKER?

Un contenedor es una instancia de una imagen, la cual tiene que estar creada con anterioridad. Así mismo un contenedor es muy parecido a una máquina virtual, pero con la peculiaridad de que comparte el mismo kernel que el host en el que se encuentra (esto reduce los recursos utilizados, ya que emular el sistema operativo es lo que consume más recursos). Gracias a lo mencionado con anterioridad, los contenedores ofrecen un mayor rendimiento que las máquinas virtuales en ciertos entornos, por ejemplo, si tienes un entorno con 10 servicios montados en Linux, es más rentable tener un único host Linux con 10 contenedores y cada uno con el servicio correspondiente, que 10 máquinas virtuales Linux y en cada máquina virtual el servicio que le corresponda. Eso si tienes un host tipo Linux, solo puedes crear contenedores Linux, ya sea Debian, RedHat, Ubuntu, Suse, etc. Eso sí, si tienes un host de tipo Linux, no puedes crear contenedores Windows.
